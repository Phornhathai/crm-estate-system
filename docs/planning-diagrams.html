Build Order — 8 Steps (As Built)
1
Project Setup
สร้าง Next.js project + ติดตั้ง dependencies ทั้งหมด
# สร้าง project (ถ้ายังไม่มี) npx create-next-app@latest crm-project \ --typescript --tailwind --app --src-dir # ติดตั้ง dependencies หลัก npm install next-auth prisma @prisma/adapter-libsql @libsql/client dotenv # ติดตั้ง dev dependencies npm install -D tsx
✅ ใช้ npm เท่านั้น (ไม่ใช่ yarn)
2
Prisma Schema + Config
ตั้งค่า Prisma 7 — แยก config ออกจาก schema
# prisma/schema.prisma generator client { provider = "prisma-client" # ⚠️ ไม่ใช่ "prisma-client-js" output = "../src/generated/prisma" } datasource db { provider = "sqlite" url = env("DATABASE_URL") # ⚠️ Prisma 7: ห้ามใส่ url ที่นี่ } # prisma.config.ts — URL อยู่ที่นี่แทน export default defineConfig({ schema: "prisma/schema.prisma", migrations: { seed: "tsx prisma/seed.ts" }, datasource: { url: process.env["DATABASE_URL"] }, })
3
Generate Client + Migrate + Seed
สร้าง Prisma Client, migrate database, ใส่ข้อมูลตัวอย่าง
npx prisma generate npx prisma migrate dev --name init npx prisma migrate dev --name seed # Seed data: 1 Owner + 2 Sale + 3 Leads + 3 FollowUps # owner@crm.com / sale1@crm.com / sale2@crm.com → password: password123 # seed.ts ต้องใช้ adapter (Prisma 7 requirement) const adapter = new PrismaLibSql({ url: process.env.DATABASE_URL }) const prisma = new PrismaClient({ adapter } as any)
⚠️ ใช้ tsx รัน seed (ไม่ใช่ ts-node) เพราะ generated client ใช้ ESM
4
Auth Setup (NextAuth v4)
ตั้งค่า NextAuth + JWT strategy + role ใน session
# src/lib/auth.ts — CredentialsProvider + JWT + role # src/app/api/auth/[...nextauth]/route.ts — export handler # src/types/index.ts — augment Session type ให้มี id + role # .env DATABASE_URL="file:./dev.db" NEXTAUTH_SECRET="crm-secret-key-change-in-production" NEXTAUTH_URL="http://localhost:3000"
5
Proxy + Layout (Next.js 16 Breaking Changes)
Route guard + App shell — ต้องแก้ไขตาม Next.js 16
# ⚠️ Next.js 16: middleware.ts → proxy.ts # src/proxy.ts export async function proxy(request: NextRequest) { ... } export const config = { matcher: ['/crm/:path*', '/login'] } # ⚠️ SessionProvider ใน Server Component ไม่ได้ # ต้องสร้าง src/components/Providers.tsx ('use client') 'use client' export default function Providers({ children }) { return <SessionProvider>{children}</SessionProvider> } # src/app/(dashboard)/layout.tsx ใช้ Providers แทน SessionProvider โดยตรง
6
API Routes (Route Handlers)
สร้าง API สำหรับ Lead + FollowUp — ระวัง params เป็น Promise
# src/app/api/leads/route.ts → GET (list) + POST (create) # src/app/api/leads/[id]/route.ts → GET + PATCH + DELETE # src/app/api/leads/[id]/followups/route.ts → GET + POST # ⚠️ Next.js 16: params เป็น Promise export async function GET( _req: NextRequest, { params }: { params: Promise<{ id: string }> } ) { const { id } = await params # ⚠️ ต้อง await! }
7
Pages — CRM UI
หน้า Login, Dashboard, Lead List, Lead Detail + FollowUp
# src/app/(auth)/login/page.tsx → 'use client', signIn() # src/app/(dashboard)/crm/dashboard/page.tsx → Server Component, role-based # src/app/(dashboard)/crm/leads/page.tsx → Lead list # src/app/(dashboard)/crm/leads/[id]/page.tsx → Lead detail + FollowUp # import Prisma client (ต้องระบุ /client ต่อท้าย) import { PrismaClient } from '@/generated/prisma/client' # ↑ ไม่ใช่ '@prisma/client'
8
Test + Verify
รัน dev server, ทดสอบ login ทั้ง 2 roles, ตรวจสอบ feature ครบ
npm run dev # http://localhost:3000 # Test checklist: ✓ Login: owner@crm.com / sale1@crm.com / sale2@crm.com ✓ Sale: เห็นเฉพาะ Lead ของตัวเอง ✓ Owner: เห็น Lead ทุกรายการ + ปุ่มลบ ✓ Dashboard: Sale เห็นสถิติตัวเอง / Owner เห็นรวม ✓ FollowUp: บันทึกและแสดงประวัติได้ ✓ Status toggle: เปลี่ยนสถานะ Lead ได้
